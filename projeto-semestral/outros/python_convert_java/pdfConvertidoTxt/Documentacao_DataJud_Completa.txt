Documentação de Alternativas de Busca Processual (Completa)

1. API Pública DataJud
- Autenticação: APIKey no cabeçalho 'Authorization'.
- Endpoint base: https://api-publica.datajud.cnj.jus.br/api_publica_<tribunal>/_search
- **Exemplos de uso:**
  1) Busca por Número de Processo:
     {
       "query": {
         "match": { "numeroProcesso": "00008323520184013202" }
       }
     }
  2) Filtro por Classe e Órgão Julgador:
     {
       "query": {
         "bool": {
           "must": [
             { "match": { "classe.codigo": 1116 } },
             { "match": { "orgaoJulgador.codigo": 13597 } }
           ]
         }
       }
     }
  3) Paginação com size + search_after:
     {
       "size": 100,
       "query": { /* filtro */ },
       "sort": [{ "@timestamp": { "order": "asc" }}],
       "search_after": [1681366085550]
     }
- Campos retornados principais: id, tribunal, numeroProcesso, dataAjuizamento, grau, formato, sistema, classe, assuntos, orgaoJulgador, movimentos, @timestamp.
- Limitação: não suporta busca por nome ou CPF de parte diretamente.

2. Alternativas de Busca por Nome
2.1 Nested Query (Elasticsearch):
     {
       "query": {
         "nested": {
           "path": "dadosBasicos.polo.parte",
           "query": {
             "match_phrase_prefix": {
               "dadosBasicos.polo.parte.pessoa.nome": "NOME"
             }
           },
           "inner_hits": {}
         }
       }
     }
2.2 Wildcard (.keyword):
     {
       "query": {
         "wildcard": {
           "dadosBasicos.polo.parte.pessoa.nome.keyword": {

             "value": "*NOME*"
           }
         }
       }
     }
2.3 Download MTD + Filtro Local:
     GET https://api-publica.datajud.cnj.jus.br/mtd/<tribunal>.json
     // Filtrar em JS usando .filter(...) por nomeInput

3. SOAP MNI (Modelo Nacional de Interoperabilidade)
- WSDL (exemplo homologação):
  https://www.cnj.jus.br/.../servico-intercomunicacao-2.2.2.wsdl
- Operação: consultarProcesso
  Envelope SOAP:
    <ser:consultarProcesso>
      <ser:filtro>
        <ser:parte>
          <ser:nomeParte>JOÃO DA SILVA</ser:nomeParte>
        </ser:parte>
      </ser:filtro>
    </ser:consultarProcesso>
- Retorno: XML com lista de processos e metadados.

4. API Codilo (Terceiros)
- REST/JSON para consulta por nome, CPF, número:
  GET https://api.codilo.com.br/v1/processos?tribunal=TJSP&parte=JOÃO%20DA%20SILVA
- Autenticação: Bearer <token>
- Suporte a filtros: parte, cpf, classe, datas, órgão julgador.

5. Considerações Finais
- Escolha entre:
  (cid:127) DataJud REST (número/processo, filtros oficiais).
  (cid:127) Elasticsearch avançado (nested, wildcard).
  (cid:127) SOAP MNI (consulta oficial por parte).
  (cid:127) Terceiro REST (Codilo, etc.).
- Balancear facilidade de uso, performance, privacidade e compliance LGPD.
